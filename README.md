# 智学闭环

基于 AI 的学习资料智能测评与个性化复习工具。项目将“资料输入、知识点提取、智能出题、在线测评、错因诊断、复习计划、强化练习、学习报告”串成完整学习闭环，适合黑客松和比赛路演展示。

第二版重点增强了“可换资料演示”能力：系统会根据用户输入的学习资料半动态提取知识点并生成题目，不再只能演示固定的人工智能示例内容。

第三版进一步升级为多格式资料输入 Demo：支持 TXT、PDF、Word `.docx`、PPT `.pptx` 在浏览器端本地解析，上传后自动提取文本、展示文件元信息，并进入同一套学习闭环。

第四版重点修复题目质量：Mock 模式不再把知识点标题或碎片词硬拼成选项，而是生成完整题干、完整选项、合理干扰项、解析和来源依据；同时接入 OpenAI / DeepSeek / Qwen API，可在本地 Demo 中切换真实大模型生成。

第五版进行 UI 重构：整体从偏黑色科技风调整为柔和、清爽、专业的教育 SaaS 风格，重点优化首页、资料导入中心、在线测评、结果分析、错因诊断、复习计划和报告预览的可读性与路演观感。

第五版同时补充了页面内 AI 接入入口：顶部“接入真实 AI”按钮可直接配置 OpenAI、DeepSeek、Qwen 的 API Key、模型和 Base URL，也可以一键切回 Mock 演示模式。

第六版升级为中国考试训练模式：系统会识别学科，按考点、题型、母题、变式、得分点、标准步骤和易错项生成训练内容；复习计划和二次强化练习不再停留在泛泛建议。

## 核心功能

- 资料输入：支持粘贴文本、上传 `.txt` / `.pdf` / `.docx` / `.pptx` 文件，并内置《人工智能基础概念》示例资料。
- 知识点提取：模拟 AI 从学习资料中提取知识点、重要程度、掌握目标和考查方式。
- 智能出题：根据知识点动态生成单选题、判断题和简答题。
- 中国考试训练：识别学科类型，生成基础概念题、公式套用题、条件辨析题、易错判断题、材料分析题、变式迁移题和综合解答题。
- 题目质量检查：每道题必须有完整题干、完整选项、正确答案、解析和来源依据。
- 在线测评：支持真实交互答题、未答完提交确认、一键优秀答卷、一键含错答卷，以及简答题图片上传/拍照识别。
- 步骤评分：简答题按标准步骤和得分点给部分分，展示命中得分点、缺失得分点和反馈。
- 错因诊断：根据错题生成概念混淆、关键词遗漏、场景判断错误等诊断。
- 个性化复习：生成 3 天复习计划和薄弱知识点强化练习。
- 教师端预览：展示上传课件、生成随堂测、班级掌握率、薄弱排行和分层任务。
- 报告导出：生成 Markdown 学习报告，支持复制、下载和浏览器打印 / 导出 PDF。

## 第二版新增功能

- 半动态 Mock AI：`src/services/aiService.ts` 会拆分段落和句子，识别“是、包括、分为、特点、作用、原因、影响、应用、区别、流程、原则”等关键词，自动生成 4-8 个知识点。
- 动态题目生成：题目围绕当前资料的知识点生成，不再固定为 AI 基础概念题。
- 路演答卷按钮：答题页提供“一键生成优秀答卷”和“一键生成含错答卷”，便于快速展示高分报告或错因诊断链路。
- 错题证据增强：结果页展示用户答案、正确答案、答案解析、对应知识点和得分情况。
- 首页路演表达增强：增加学习痛点、解决方案、应用场景和教师端扩展预览。
- 步骤条可回看：已完成步骤可以点击返回，未完成步骤保持不可点击。

## 第三版新增功能

- 支持 PDF / Word(.docx) / PPT(.pptx) / TXT 多格式资料上传。
- PDF 使用 `pdfjs-dist` 在浏览器端解析文本，并展示页数。
- Word 使用 `mammoth` 提取 `.docx` 纯文本。
- PPT 使用 `jszip` 解析 `.pptx` 内部 slide XML，并展示幻灯片页数。
- 支持文件名、文件类型、文件大小、提取字数、页数等元信息展示。
- 支持解析失败提示，页面不会白屏。
- 扫描版 PDF 暂不支持 OCR。
- 老版 `.doc` / `.ppt` 暂不支持。

## 第四版新增功能

- 重写 Mock 出题逻辑，避免出现“标点符号在高职高考中 / 分）形式知识点稳定”这类碎片选项。
- 知识点新增 `sourceEvidence` 和 `keywords`，题目新增 `sourceEvidence` 和 `qualityScore`。
- 新增题目质量校验：题干长度、单选 4 个完整选项、正确答案在选项中、解析和来源依据完整。
- 新增真实 API 调用：支持 OpenAI Responses API，以及 DeepSeek / Qwen 的 OpenAI-compatible Chat Completions。
- API 未配置或请求失败时自动回退 Mock，不会白屏。
- 页面顶部显示当前模式：Mock 演示模式 / OpenAI API / DeepSeek API / Qwen API / API 回退 Mock。

## 第五版视觉优化

- 整体改为柔和浅色教育 SaaS 风格，减少大面积暗色和低对比度文字。
- 首页突出“资料上传 → 智能测评 → 错因诊断 → 个性复习”的闭环价值。
- 资料上传页升级为文件导入中心，清晰展示 TXT / PDF / DOCX / PPTX 能力和解析元信息。
- 测评题目页和在线答题页更像真实考试系统，题卡、选项、题型标签更清晰。
- 一键优秀答卷、一键含错答卷被收纳为演示辅助功能，不抢主流程。
- 结果页强化总分、掌握率、薄弱点、错题证据和来源依据。
- 错因诊断页按错因类型、用户答案、正确理解、复习建议分层展示。
- 报告导出页升级为正式报告预览界面，支持 Markdown 下载、复制和打印 / 导出 PDF。
- 教师端预览改为更真实的数据看板，展示班级掌握率、薄弱排行和分层任务。
- 顶部新增“接入真实 AI”按钮，路演时可以直接在页面内切换 OpenAI、DeepSeek、Qwen 或 Mock。

## 第六版考试训练模式

- 考点识别：知识点会附带学科类型、公式/规则、常见误区和关键方法。
- 题型匹配：按中国考试训练策略匹配基础概念题、公式套用题、条件辨析题、易错判断题、材料分析题、变式迁移题和综合解答题。
- 母题生成：数学/物理/化学类题目强调公式、条件、步骤和规范结论。
- 变式训练：二次强化题从错题题型和薄弱知识点出发，生成同类换条件、换数值或换语境题。
- 步骤评分：简答题/解答题按标准步骤、得分点和关键公式给部分分。
- 错因修复：结果页展示缺失得分点、常见误区和推荐强化训练方向。

## 简答题拍照上传

- 简答题/解答题支持上传图片，移动端可直接拍照。
- 图片仅用于浏览器预览和提交给视觉模型识别，不会永久保存。
- 当前图片识别优先支持 OpenAI vision 能力；不支持 vision 的模型会提示手动输入。
- 识别结果会自动填入答案框，学生可以继续编辑后再提交。

## 二次强化训练

- 强化题标题为“二次强化训练：同类变式与错因修复”。
- 默认隐藏标准答案，学生需要先作答，再点击“查看答案”。
- 每题包含标准答案、标准步骤、得分点、常见误区和来源依据。
- 支持“我已掌握 / 仍需复习”标记，适合路演展示错因修复闭环。

## 具体复习计划

- 每天包含今日目标、必背公式/定义、例题任务、强化练习、常见误区、自测标准和勾选清单。
- 数学类资料会突出公式记忆、条件识别、步骤推导、符号规范和变式训练。
- 语文类资料会突出规则识别、语境判断、病因分析和材料依据。

## 页面内接入真实 AI

1. 启动项目后，点击顶部右侧的“接入真实 AI”按钮。
2. 选择 OpenAI、DeepSeek 或通义千问 Qwen。
3. 填写 API Key、模型名称和 Base URL。
4. 点击“保存配置”。
5. 回到演示流程，重新点击“开始分析”或“生成测评题目”，系统会优先调用真实 API。
6. 如果请求失败或 Key 为空，系统会自动回退 Mock，不会影响演示。

安全提醒：页面内配置会保存到当前浏览器的 `localStorage`，只适合本地比赛 Demo。正式上线必须使用后端代理保护 API Key。

## 一键演示流程

1. 打开首页，先讲“学习痛点、解决方案、应用场景、教师端扩展”。
2. 点击“开始体验”，上传一个 PDF / Word / PPTX 文件。

## 路演推荐操作顺序

推荐用“先价值、再闭环、最后扩展”的顺序：

1. 首页：先讲学生复习痛点，再讲资料驱动闭环。
2. 资料输入：上传一个 PDF / Word / PPTX 文件。
3. 文件解析：展示提取文本和文件元信息。
4. 知识点页：强调 4-8 个知识点来自材料，而不是固定题库。
5. 答题页：使用“一键生成含错答卷”，快速制造诊断样本。
6. 结果页：展示可解释评分依据。
7. 诊断和计划：强调错题不是终点，而是复习路径的起点。
8. 教师端预览：说明可扩展为班级学情分析和分层作业平台。

## 环境变量

复制 `.env.example` 为 `.env` 后按需配置：

```bash
VITE_AI_PROVIDER=mock

VITE_OPENAI_API_KEY=
VITE_OPENAI_MODEL=gpt-4.1-mini
VITE_OPENAI_BASE_URL=https://api.openai.com/v1

VITE_DEEPSEEK_API_KEY=
VITE_DEEPSEEK_MODEL=deepseek-chat
VITE_DEEPSEEK_BASE_URL=https://api.deepseek.com/v1

VITE_QWEN_API_KEY=
VITE_QWEN_MODEL=qwen-plus
VITE_QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
```

默认 `VITE_AI_PROVIDER=mock`，不需要后端和数据库即可运行完整 Demo。

## 生成模式

当前支持两种生成模式：

- Mock 演示模式：默认模式，本地规则生成知识点、题目、诊断，适合离线路演。
- API 真实生成模式：配置 OpenAI / DeepSeek / Qwen Key 后，由外部大模型生成知识点、题目和诊断。

## 如何开启 API

1. 复制环境变量文件：

```bash
copy .env.example .env
```

2. 选择 Provider，例如 DeepSeek：

```bash
VITE_AI_PROVIDER=deepseek
VITE_DEEPSEEK_API_KEY=你的 Key
```

3. 重新启动开发服务器：

```bash
npm run dev
```

4. 打开页面顶部状态标签，确认显示“DeepSeek API 已启用”等真实 API 状态。

如果没有配置 Key，或 API 请求失败，系统会自动回退到 Mock，并在浏览器控制台输出错误。

## API Key 安全提醒

`VITE_` 开头的环境变量会被 Vite 注入浏览器端代码，任何访问页面的人都可能看到这些变量。当前接入方式只适合本地比赛 Demo 或临时演示。正式上线必须通过后端代理保护 API Key，不要把真实密钥直接暴露在前端。

## 题目质量标准

系统会校验生成题目是否满足：

- 题干完整，长度足够，不是碎片词。
- 单选题必须有 4 个完整句子选项。
- 干扰项合理但错误，不能使用“无关干扰项”“资料背景信息”等假选项。
- 正确答案必须在选项中。
- 每题必须有答案解析。
- 每题必须有来源依据 `sourceEvidence`。
- 题目和选项不能以 `/`、`、`、`）`、`.` 等明显残片开头。

## 接入真实大模型 API 的方式

真实 API 入口保留在 `src/services/aiService.ts`：

1. 修改 `.env`：

```bash
VITE_AI_PROVIDER=openai
VITE_OPENAI_API_KEY=你的 Key
```

2. 请求封装位于 `src/services/llmClient.ts`。
3. Prompt 模板位于 `src/services/promptTemplates.ts`。
4. 模型必须返回结构化 JSON，字段保持与 `KnowledgePoint`、`QuizQuestion`、`DiagnosisItem` 等类型一致。
5. Mock 会作为离线 fallback，避免比赛现场网络不稳定影响演示。

## 比赛路演介绍话术

“智学闭环”不是单点式 AI 出题工具，而是面向学生自主复习的完整闭环系统。用户只需要上传自己的学习资料，系统就能自动提取知识点、生成测评题目、完成在线评分，并根据错题定位薄弱知识点，进一步生成复习计划和二次强化练习。它把传统复习中分散的资料整理、刷题、批改、错题复盘和计划制定整合到一个轻量化 Web 工具中，既适合学生个人使用，也可以扩展到教师端进行班级学情分析。
