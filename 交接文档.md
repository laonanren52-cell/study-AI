## 当前状态 —— 核心问题

**点击"提交生成题目"后，主流程已能跳转到智能测评页，但生成的题目质量极差（"理解并掌握基本概念""以上都是"这类垃圾题）。**

## 项目架构

### 关键文件
- src/App.tsx - 主组件，handleGenerateQuiz 是核心流程入口
- src/services/fallbackQuestionFactory.ts - generateFallbackQuestionsFromBlueprints 本地题库出题
- src/services/questionGenerationOrchestrator.ts - AI 出题调度器
- src/services/materialTopicService.ts - inferMaterialProfile / inferMaterialTopic 资料主题识别
- src/services/questionTopicVerifier.ts - verifyQuestionTopicAlignment 主题校验
- src/services/llmClient.ts - callLLMJson，AI 调用（30s timeout）
- src/components/KnowledgePointList.tsx - 知识学习页，包含"提交生成题目"按钮
- src/components/QuizTaking.tsx - 智能测评答题页

### 出题流程
handleGenerateQuiz -> [SKIP AI] -> generated = [] -> 进入 fallback 区块
  -> 构建 pseudoBlueprints（基于知识点 + materialTopic）
  -> generateFallbackQuestionsFromBlueprints(blueprints, [], settings, fallbackTopic)
  -> 内部：为每个 blueprint 调用 buildMathFallbackQuestion 等策略生成题
  -> 内部：verifyQuestionTopicAlignment 校验题目是否符合资料主题
  -> 如果 topicTag === "通用知识"，跳过校验；否则过滤掉不匹配的题
  -> 返回 filtered questions
  -> setQuestions + goToStep('quiz')

### 问题的关键链路
1. generateFallbackQuestionsFromBlueprints 内部生成了题
2. 但如果 materialTopic.topicTag !== "通用知识" 且 verifyQuestionTopicAlignment 对所有题返回 false
3. 所有题被过滤掉，返回空数组
4. generated.length === 0
5. 接下来的代码走到 goToStep('quiz') 但 setQuestions 的已经是空数组
6. 用户看到测评页但没有题

## 建议修复方向

### 方案 A（推荐）：让 materialTopic 的 topicTag 变成"通用知识"
在 fallback 区块中，如果 inferMaterialTopic 返回的 topicTag 不是"通用知识"，
或者 allowedTemplateIds 为空，手动构建一个 topicTag="通用知识" 的 fallbackTopic。
这样 verifyQuestionTopicAlignment 会跳过校验，generateFallbackQuestionsFromBlueprints
生成的所有题会直接通过。

具体：在 handleGenerateQuiz fallback 区块的这段代码之后：
`
const fallbackTopic = inferMaterialTopic(material.content, kpList, subjectType);
`
加上：
`
// 如果 topic 不明确，强制使用"通用知识"跳过主题校验
if (!fallbackTopic || !fallbackTopic.allowedTemplateIds || !fallbackTopic.allowedTemplateIds.length) {
  fallbackTopic.allowedTemplateIds = ['math-01','math-02']; // 或其他有效模板
  fallbackTopic.topicTag = '通用知识';
}
`

### 方案 B：直接调用 reinforceFactory 出题
不经过 generateFallbackQuestionsFromBlueprints，直接使用 src/services/fallbackQuestionFactory.ts
中更底层的 buildMathFallbackQuestion 等函数。这些函数针对不同学科（数学/化学/物理等）
生成实际的考点题目，有具体的数值和公式，不是"理解并掌握"这种泛泛题。
